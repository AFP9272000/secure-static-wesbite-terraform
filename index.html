Secure Static Site
Terraform • AWS

Private S3 origin. Public edge. Secure by design.

Static site served only through CloudFront using Origin Access Control, protected by AWS WAF, and provisioned with Terraform. Direct S3 access is blocked by policy.

S3 Private CloudFront (OAC) WAF Managed Rules HTTPS Least Privilege

Architecture

  • S3 bucket: private, encrypted, public access blocked, ACLs disabled.
  • CloudFront: global edge delivery, OAC signed requests.
  • AWS WAF: managed rule sets + rate limiting at the edge.
  • Terraform: one stack builds the infrastructure end-to-end.

Validation

curl -I https://<bucket>.s3.<region>.amazonaws.com/index.html
→ 403 Forbidden (blocked as intended)

curl -I https://<cloudfront-domain>/
→ 200 OK (served through CloudFront)

200 via CloudFront, 403 via S3 = correct behavior.

Deployment Notes

# sync site contents
aws s3 sync ./site s3://<bucket> --delete

# invalidate CloudFront cache
aws cloudfront create-invalidation \
  --distribution-id <id> --paths "/*"
